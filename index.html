PERINTAH TEKS JARINGAN DI COMMAND PROMP

(CMD) adalah instruksi berbasis teks untuk ping untuk mengecek konektivitas IP config jalur data, netstat untuk statistik koneksi, dan NS lookup untuk info DNS yang semuanya digunakan untuk pemecahan masalah dan konfigurasi jaringan di Windows PERINTAH JARINGAN DASAR DAN FUNGSINYA IP config: menampilkan konfigurasi IP (alamat IP, subnet mask, gateway default) ipconfig/all: menampilkan konfirmasi detail lengkap ipconfig / flushdns: besok pulang kan membersihkan cache DNS IP config /relese & renew: melepas dan mendapat IP baru *ping [alamat]*: menguji konektivitas ke host lain (misal : ping google.com) *tracert[alamat]*: melacak jalur paket data ke tujuan (misal: tracert Google.com) netstat: menampilkan koneksi jaringan aktif, tabel routing dan statistik nslookup [nama_domain]: memeriksa informasi DNS untuk nama domain (misal: nslookup google.com) route: menampilkan atau memanipulasi tabel routing IP hostname: menampilkan nama komputer anda cara membuka CMD tekan tombol Windows + R, ketik CMD, lalu tekan enter klik start, ketik CMD, dan pilih untuk membukanya contoh penggunaan cek koneksi ke Google: ketik ping google.com lalu enter. jika berhasil, akan muncul respons waktu (Time) dan byte lihat alamat IP Anda: ketik ipconfig lalu enter cari "ipv4 address" cari tahu alamat IP dari domain: ketik nslookup www.facebook.com lalu enter

IP address

IP address => merupakan alamat dalam jaringan yang terhubung antara client 1 dan 2 dan seterusnya di dalam IP address client terdapat gateway, apa itu gateway=>lalu lintas atau jalan di dalam IP address terdapat juga subnet mask subnet mask =>seolah kendaraan IP address adalah nomor penumpan gateway jalannya atau lalu lintas (kabel dan nirkabel) alamat adalah berupa angka yang dikatakan dalam klien di dalam IP address itu terdapat kelas : - Kelas A → 1.0.0.0 sampai 126.0.0.0 - Kelas B → 128.0.0.0 sampai 191.255.0.0 - Kelas C → 192.0.0.0 sampai 223.255.255.0 - kelas D → 224.0.0.0 sampai 239.255.255.255 - Kelas E → 240.0.0.0 sampai 255.255.255.255

aplikasi virtual

#VirtualBox
*cara kerja: VirtualBox bekerja dengan membuat virtual (vm) yang meniru perangkat keras komputer setiap VM bisa menjalankan sistem operasi berbeda di atas host OS.
*fungsi dan manfaat:
-menjalankan beberapa OS sekaligus (Windows, Linux, macos,)
-testing software tanpa resiko ke sistem utama
-simulasi jaringan virtual
-isolasi lingkungan kerja
*kegunaan: cocok untuk developer, pelajar, dan IT ada yang butuh uji coba sistem atau aplikasi.
*konfigurasi:
-install VirtualBox di hots OS
-buat VM (virtual machine) lalu tentukan RAM, CPU, dan storage
-pasang iso OS yang diinginkan
-atur jaringan (N A T, BRIDGE, HOTS-ONLY)

#vmware
*cara kerja: PMR menggunakan teknologi virtualisasi untuk membagi sumber daya server fisik menjadi beberapa VM, setiap VM berjalan seolah-olah memiliki hardware sendiri.
*fungsi dan manfaat:
-menjalankan banyak OS dalam satu perangkat
-mendukung skala besar untuk mendata center dan cloud
-mempermudah manajemen server dan aplikasi
-efisiensi biaya dan fleksibilitas tinggi
*kegunaan: digunakan perusahaan untuk cloud computing server virtualization dan pengelolaan infrastruktur IT
*konfigurasi:
-install vmware workstation/ESXI
-buat VM lalu tentukan spesifikasi hardware virtual
-pasang OS via iso
-atur integrasi dengan vsphere/v center untuk manajemen skala besar

#proxmox
*cara kerja: rock smoke adalah platform open-sourseberbasis web untuk mengelola PM dan container mendukung kvm (virtualisasi penuh) dan lxc (container)
*fungsi dan manfaat:
-open-source dan gratis
-mendukung cluster ring dan high availability
-firewall bawaan dan integrasi dengan storage
*kegunaan: ideal untuk perusahaan kecil hingga data center besar yang dibutuhkan solusi hemat biaya tapi powerful
*konfigurasi:
-install proxmox ve di server fisik
-akses via web interface
-buat PM atau kontainer dan tentukan resource
-atur storage, jaringan, dan clostering jika diperlukan
